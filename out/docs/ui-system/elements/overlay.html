Overlay

Overlay is a component to be used for dropdowns, popovers, tooltips, or even modal windows with triggers.
// import { Overlay, Element } from '@vitus-labs/elements'


const Button = (props) => {
  const { active } = props


  return (
    <Element
      {...props}
      label="Click on me"
      tag="button"
      style={{
        background: active ? '#86C232' : 'rgba(134, 194, 50, 0.4)',
        border: '1px solid #86C232',
        padding: '8px 16px',
        color: '#fff',
        borderRadius: 4,
        outline: 'none',
      }}
    />
  )
}


const Content = (props) => (
  <Element
    style={{
      backgroundColor: '#fff',
      color: '#000',
      padding: 8,
      borderRadius: 8,
      zIndex: 2,
    }}
    {...props}
  >
    Some content here
  </Element>
)


// ↓↓↓ Overlay ↓↓↓
render(
  <Overlay trigger={Button} refName="innerRef" offsetY={8}>
    <Content />
  </Overlay>
)

Using Overlay

There are several ways of rendering text within Text component. So let's take a look at them.

Creating a dropdown

First of all, you can use Text and pass children.
// import { Overlay, Element } from '@vitus-labs/elements'


const Button = (props) => {
  const { active } = props


  return (
    <Element
      {...props}
      label="Click on me"
      tag="button"
      style={{
        background: active ? '#86C232' : 'rgba(134, 194, 50, 0.4)',
        border: '1px solid #86C232',
        padding: '8px 16px',
        color: '#fff',
        borderRadius: 4,
        outline: 'none',
      }}
    />
  )
}


const Content = (props) => (
  <Element
    contentDirection="rows"
    style={{
      backgroundColor: '#fff',
      color: '#000',
      padding: 8,
      borderRadius: 8,
      zIndex: 2,
    }}
    {...props}
  />
)


const Item = ({ label }) => (
  <Element
    label={label}
    style={{
      width: 200,
      height: 40,
      borderBottom: '1px solid rgba(134, 194, 50, 0.4)',
    }}
  />
)


// ↓↓↓ Overlay ↓↓↓
render(
  <Overlay trigger={Button} refName="innerRef" offsetY={8}>
    <Content>
      <Item label="Item A" />
      <Item label="Item B" />
      <Item label="Item C" />
      <Item label="Item D" />
    </Content>
  </Overlay>
)

Creating a tooltips

For shorter texts, or inline texts sometimes it's nicer to write inline prop. If you prefer using it this way, then label prop could be used. In case of passing both children and label, then children will have a priority and label will be ignored.
// import { Overlay, Element } from '@vitus-labs/elements'


const Button = (props) => (
  <Element
    {...props}
    label="Hover on me"
    tag="button"
    style={{
      background: 'rgba(134, 194, 50, 0.4)',
      border: '1px solid #86C232',
      padding: '8px 16px',
      color: '#fff',
      borderRadius: 4,
      outline: 'none',
    }}
  />
)


const Content = (props) => (
  <Element
    style={{
      backgroundColor: '#fff',
      color: '#000',
      padding: 8,
      borderRadius: 8,
      zIndex: 2,
    }}
    {...props}
  >
    Some content here
  </Element>
)


// ↓↓↓ Overlay ↓↓↓
render(
  <Overlay
    trigger={Button}
    refName="innerRef"
    openOn="hover"
    closeOn="hover"
    align="bottom"
    alignX="center"
    offsetY={8}
  >
    <Content />
  </Overlay>
)

Overlay Properties

These are props to which can be passed to Overlay
NameTypeDefault valueDescription
childrenReactNodeA content to be rendered when Overlay is open.
triggerComponentTypeAn element to be used to trigger opening an overlay element.
triggerRefNamestringA key name how a ref should be passed to trigger component
contentRefNamestringA key name how a ref should be passed to content component
DOMLocationHTMLElementIt uses in the background React Portal, so you can define where in the DOM the Overlay content should be attached
isOpenbooleanfalseDefines whether it should be open by default or not
openOnclick hover manualclickDefines on which action it should get opened
closeOnclick triggerClick hover manualclickDefines on which action it should get closed
typedropdown tooltip popover modaldropdownDefines type of your overlay window and based on it it defines its behavior
refNamestringrefOverlay component access ref to directly mutate styles when calculation position to prevent re-renders. It's being used for both trigger, and children element at the same time. Your components must accept refs with the same naming.
positionfixed absolute relative staticfixedJust a CSS property to be used for styling overlay element. You can change it's behavior when necessary. Accepts all available CSS position values.
aligntop left right bottombottomOn which side it should be aligned to trigger
alignXleft center rightleftIt will calculate it's position horizontally based on align value. This will take an effect only when align is set to either top, or bottom
alignYtop center bottombottomIt will calculate it's position horizontally based on align value. This will take an effect only when align is set to either left, or right
offsetXnumber0Horizontal offset value from trigger element
offsetYnumber0Vertical offset value from trigger element
throttleDelaynumber200An optimization value, being used for calculations.
