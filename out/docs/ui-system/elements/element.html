Element

Element is the smallest possible unit to be used for creating components. By default it renders div. You can render any valid HTML tag by passsing it as a prop.Element is by default inline-flex. You can set flex by using boolean block prop. This prop also stretches the element to width or height of 100% based on Element direction.By default, Element also checks if tag passed as a prop is type of inline HTML tag like button or span. If so, sub-elements helpers rendered inside (beforeContent, content and afterContent wrappers) are by default rendered as span element to make sure it renders valid HTML.

Examples

Horizontal example

// import { Element } from '@vitus-labs/elements'

const BeforeAfter = () => <Element css={beforeAfterStyles} />

const Content = () => <Element css={contentStyles} />

const beforeAfterStyles = `
  ${defaultCss};
  width: 40px;
  height: 100%;
`

const contentStyles = `
  ${defaultCss};
  width: 100%;
  height: 100%;
`

const styles = `
  ${defaultCss};
  width: 100%;
  min-width: 320px;
  height: 240px;
  padding: 16px;
`

//  Element 
render(
  <Element
    gap={16}
    content={Content}
    css={styles}
    beforeContent={BeforeAfter}
    afterContent={BeforeAfter}
  />
)

Render children

To render children inside component you can pass content as children, content, or label prop. label and content props are shorthands which might look nicer for inline elements. If more of those props are passed, children will have the highest priority to be rendered. label and/or content will be ignored. In other words, rendering priorities are children content label. Any of those prop names can be used based on your prefeference.
// import { Element } from '@vitus-labs/elements'

// ------------------------------------
// helper styles
// ------------------------------------
const beforeAfterStyles = `
  ${defaultCss};
  width: 40px;
  height: 100%;
`

const contentStyles = `
  ${defaultCss};
  width: 100%;
  height: 100%;
`

const styles = `
  ${defaultCss};
  width: 100%;
  min-width: 240px;
  height: 48px;
  padding: 16px;
  margin-bottom: 12px;
`

// ------------------------------------
// helper elements
// ------------------------------------
const BeforeAfter = () => <Element css={beforeAfterStyles} />

const Content = () => <Element css={contentStyles} />

//  Element 
render(
  <>
    {/* used content prop here */}
    <Element
      gap={16}
      content={Content}
      css={styles}
      beforeContent={BeforeAfter}
      afterContent={BeforeAfter}
    />

    {/* used label prop here */}
    <Element
      gap={16}
      label={Content}
      css={styles}
      beforeContent={BeforeAfter}
      afterContent={BeforeAfter}
    />

    {/* used children prop here */}
    <Element
      gap={16}
      content={Content}
      css={styles}
      beforeContent={BeforeAfter}
      afterContent={BeforeAfter}
    >
      <Content />
    </Element>
  </>
)

Before & After content

For extending the Element component, beforeContent and/or afterContent can be passed. This may be also kind of alternative to using :before and/or :after pseudo elements.This might be useful especially in cases like buttons with loading icon or input wrappers with icons or anything similar.In the examples below can be found several examples of how beforeContent and/or afterContent can be passed as a prop. This applies to label and content props as well.
// import { Element } from '@vitus-labs/elements'

const beforeAfterStyles = `
  ${defaultCss};
  width: 40px;
  height: 100%;
`

const contentStyles = `
  ${defaultCss};
  width: 100%;
  height: 100%;
`

const styles = `
  ${defaultCss};
  width: 100%;
  min-width: 240px;
  height: 48px;
  padding: 16px;
  margin-bottom: 12px;
`

const BeforeAfter = () => <Element css={beforeAfterStyles} />

const Content = () => <Element css={contentStyles} />

{
  /*
  beforeContent - renders box on the leftt side
  afterContent - renders box on the right side
*/
}

//  Element 
render(
  <>
    {/* [1] Function component as prop */}
    <Element
      gap={16}
      content={Content}
      css={styles}
      beforeContent={BeforeAfter}
      afterContent={BeforeAfter}
    />

    {/* [2] Rendered component as prop */}
    <Element
      gap={16}
      content={Content}
      css={styles}
      beforeContent={<BeforeAfter />}
      afterContent={<BeforeAfter />}
    />

    {/* [3] Function as prop */}
    <Element
      gap={16}
      content={Content}
      css={styles}
      beforeContent={() => <BeforeAfter />}
      afterContent={() => <BeforeAfter />}
    />
  </>
)

Vertical & Horizontal alignment

Element can be either horizontally or vertically. Based on the alignment definition beforeContent on the left side or above the children content. Respectively, the same rule applies to afterContent (below children or on the right side). Check out the examples below to see more.For vertical aligning can be used prop direction prop which can be set to rows | inline | reverseRows | reverseInline.

Horizontal variant

Element aligns inner elements horizontally by default.

const beforeAfterStyles = `

  width: 40px;
  height: 100%;
`

const contentStyles = `
  ${defaultCss};
  width: 100%;
  height: 100%;
`

const styles = `
  ${defaultCss};
  width: 100%;
  min-width: 240px;
  height: 48px;
  padding: 16px;
  margin-bottom: 12px;
`

const BeforeAfter = () => <Element css={beforeAfterStyles} />

const Content = () => <Element css={contentStyles} />

{
  /* By default, Element is horizontal */
}

//  Element 
render(
  <Element
    gap={16}
    content={Content}
    css={styles}
    beforeContent={BeforeAfter}
    afterContent={BeforeAfter}
  />
)

Horizontal element

Even inner elements can be aligned horizontally or vertically inside the root element as well.See the examples below.

Align beforeContent horizontally

Aligning beforeContent horizontally (on axis X) is configured via beforeContentAlignX prop with accepting values left, center, right, spaceBetween, spaceAround or block.
beforeContent aligned (left, center, right)

Align content horizontally

Aligning content horizontally (on axis X) is configured via contentAlignX prop with accepting values left, center, right, spaceBetween, spaceAround or block.
children / content / label aligned (left, center, right)

Align afterContent horizontally

Aligning afterContent horizontally (on axis X) is configured via afterContentAlignX prop with accepting values left, center, right, spaceBetween, spaceAround or block.
afterContent aligned (left, center, right)

Align beforeContent vertically

Aligning beforeContent vertically (on axis Y) is configured via beforeContentAlignY prop with accepting values top, center, bottom, spaceBetween, spaceAround or block.
beforeContent aligned (top, center, bottom)

Align content vertically

Aligning content vertically (on axis Y) is configured via contentAlignY prop with accepting values top, center, bottom, spaceBetween, spaceAround or block.
children / content / label aligned (top, center, bottom)

Align afterContent vertically

Aligning afterContent vertically (on axis Y) is configured via afterContentAlignY prop with accepting values top, center, bottom, spaceBetween, spaceAround or block.
afterContent aligned (top, center, bottom)

Vertical element

Element can be aligned horizontally or vertically. So you can have beforeContent on the left side of children or above. The same applies for afterContent (below children or on the right side). Check out the examples below to see more. For vertical aligning you can use prop vertical set to true or use direction prop which can be set to rows or inline.
